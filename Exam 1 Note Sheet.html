2) Supervised Binary Classification

Core idea

Given input x, predict class y ∈ {0,1}

Decision Boundary

A surface that separates classes in feature space.

  • Linearly separable: one line/plane can separate classes.
  • Nonlinearly separable: need curved boundary or more complex model.

Confusion Matrix

Predicted Positive Predicted Negative
Actual Positive TP FN
Actual Negative FP TN

Metrics

Accuracy
Accuracy=TP+TNTP+TN+FP+FN\text{Accuracy}=\frac{TP+TN}{TP+TN+FP+FN}Accuracy=TP+TN+FP+FNTP+TN​

Precision
Precision=TPTP+FP\text{Precision}=\frac{TP}{TP+FP}Precision=TP+FPTP​

Recall
Recall=TPTP+FN\text{Recall}=\frac{TP}{TP+FN}Recall=TP+FNTP​

F1-score
F1=2⋅Precision⋅RecallPrecision+RecallF1=2\cdot \frac{\text{Precision}\cdot \text{Recall}}{\text{Precision}+\text{Recall}}F1=2⋅Precision+RecallPrecision⋅Recall​

Quick intuition

  • Precision: “When model says positive, how often is it right?”
  • Recall: “How many real positives did it catch?”
  • F1: balance between precision and recall.

3) ML Classifiers

Support Vector Machine (SVM)

Goal: maximize margin between classes.

For linear case:
f(x)=wTx+bf(x)=w^T x+bf(x)=wTx+b

Prediction:
y^=sign(wTx+b)\hat{y}=\text{sign}(w^T x+b)y^​=sign(wTx+b)

  • Works well for clear boundaries
  • Can use kernels for nonlinear separation
  • Important idea: maximum margin

K-Nearest Neighbors (KNN)

  • Classify based on the k closest points

  • Common distance:
    d(xi,xj)=∑k(xik−xjk)2d(x_i,x_j)=\sqrt{\sum_k (x_{ik}-x_{jk})^2}d(xi​,xj​)=k∑​(xik​−xjk​)2​

  • Small (k): sensitive/noisy

  • Large (k): smoother, may underfit

Decision Trees

  • Split data using feature thresholds
  • Easy to interpret
  • Can overfit if tree is too deep

Common split criteria:

  • Gini impurity
    G=1−∑ipi2G = 1 - \sum_i p_i^2G=1−i∑​pi2​

  • Entropy
    H=−∑ipilog⁡2piH = -\sum_i p_i \log_2 p_iH=−i∑​pi​log2​pi​

Random Forests

  • Ensemble of many decision trees
  • Uses bootstrap samples + random feature subsets
  • Reduces overfitting compared to one tree

4) Data Stratification / Model Evaluation

Train / Validation / Test

  • Training set: fit parameters
  • Validation set: tune hyperparameters
  • Test set: final unbiased evaluation

Overfitting

Model memorizes training data, poor generalization.

Underfitting

Model too simple, poor performance even on training data.

Holdout Validation

Single split into train/validation/test.

Cross-Validation

Repeated splits; common form is k-fold CV.

Why stratification matters

Preserve class proportions across splits, especially for imbalanced datasets.

7) Backpropagation

Core idea

Use gradients to update weights to reduce loss.

Gradient descent update:
w←w−η∂L∂ww \leftarrow w - \eta \frac{\partial L}{\partial w}w←w−η∂w∂L​

b←b−η∂L∂bb \leftarrow b - \eta \frac{\partial L}{\partial b}b←b−η∂b∂L​

Where (\eta) is the learning rate.

Chain Rule

Backprop uses chain rule through layers:
∂L∂w=∂L∂a∂a∂z∂z∂w\frac{\partial L}{\partial w}= \frac{\partial L}{\partial a} \frac{\partial a}{\partial z} \frac{\partial z}{\partial w}∂w∂L​=∂a∂L​∂z∂a​∂w∂z​

Important intuition

  • Forward pass computes prediction
  • Loss measures error
  • Backprop computes how each weight caused error
  • Optimizer updates weights

8) Neural Network Hyperparameters / Concepts

Minibatch Size

Number of samples used per weight update.

Epoch

One full pass through training set.

Stopping Criteria

Stop when:

  • validation loss stops improving
  • max epochs reached
  • early stopping triggered

Bias

Extra learnable offset:
z=wTx+bz = w^T x + bz=wTx+b

Vanishing Gradient

Gradients become tiny in deep nets, slowing learning.
Common with sigmoid/tanh in deep networks.

Update Algorithms

Basic Gradient Descent

θ←θ−η∇θL\theta \leftarrow \theta - \eta \nabla_\theta Lθ←θ−η∇θ​L

Stochastic Gradient Descent (SGD)

Uses one example (or minibatch)

Momentum idea

Uses past gradients to smooth updates

Adam

Adaptive learning rates + momentum; very common

9) Common Neural Network Layers

Fully Connected Layer

Every neuron connected to every output from previous layer.

Softmax Layer

Converts logits into class probabilities.

Dropout

Randomly zeroes some neurons during training to reduce overfitting.

Batch Normalization

Normalizes activations within a minibatch to stabilize training.

Pooling

Downsamples feature maps.

  • Max pooling: keeps strongest activation
  • Average pooling: averages values

Convolutional Layer

Applies learnable filters over local regions.

Recurrent Layer

Handles sequences by carrying hidden state over time.

Attention Layer

Lets model focus on relevant tokens/positions.

10) CNNs

Convolution

A filter/kernel slides across input.

Very general form:
(feature map)(i,j)=∑m∑nX(i+m,j+n)K(m,n)(feature\ map)(i,j)=\sum_m \sum_n X(i+m,j+n)K(m,n)(feature map)(i,j)=m∑​n∑​X(i+m,j+n)K(m,n)

What CNNs are good for

  • images
  • spatial patterns
  • local feature extraction

Typical CNN flow

Conv → activation → pooling → fully connected

Forward Pass Intuition

  • early layers detect edges/textures
  • deeper layers detect complex shapes/features

Backprop in CNNs

  • gradients flow through convolution filters too
  • each filter weight is updated based on contribution across all positions where it was used

11) RNNs

Core idea

Sequence model with hidden state.

At time (t):
ht=f(Wxhxt+Whhht−1+bh)h_t = f(W_{xh}x_t + W_{hh}h_{t-1} + b_h)ht​=f(Wxh​xt​+Whh​ht−1​+bh​)
yt=Whyht+byy_t = W_{hy}h_t + b_yyt​=Why​ht​+by​

  • Good for time series / sequences
  • Shares parameters across time

LSTM

Designed to reduce vanishing gradient problems in long sequences.

Main idea:

  • has cell state

  • uses gates:

    • forget gate
    • input gate
    • output gate

You probably want to remember why LSTMs exist:

They preserve information longer than standard RNNs.

12) Transformers

Core idea

Sequence model based on attention, not recurrence.

Encoder / Decoder

  • Encoder: reads input sequence
  • Decoder: generates output sequence

Tokenization

Break text into smaller units called tokens.

Positional Encoding

Transformers do not inherently know token order, so position is added to embeddings.

Q, K, V (Query, Key, Value)

Each token is projected into:

  • Q = what am I looking for?
  • K = what do I contain?
  • V = what information do I pass along?

Self-Attention Equation

Attention(Q,K,V)=softmax(QKTdk)V\text{Attention}(Q,K,V)=\text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)VAttention(Q,K,V)=softmax(dk​​QKT​)V

Important parts

  • (QK^T): similarity scores
  • divide by (\sqrt{d_k}): stabilizes values
  • softmax: converts to attention weights
  • multiply by (V): weighted combination of information

Types of Attention

  • Self-attention: sequence attends to itself
  • Encoder-decoder attention: decoder attends to encoder outputs

Feed-Forward / Fully Connected Layers

Applied after attention to transform representations further.

Why Transformers are powerful

  • parallelizable
  • capture long-range dependencies
  • foundation of modern LLMs

13) LLMs and Prompting

Zero-shot Prompting

Ask model to do task with no examples.

Example:

  • “Summarize this text.”

One-shot Prompting

Give one example first.

Few-shot Prompting

Give a few examples to teach desired pattern.

Chain-of-Thought Prompting

Encourage step-by-step reasoning.

Example:

  • “Explain your reasoning step by step.”

Prompt Chaining

Output of one prompt becomes input to the next.

Tree-of-Thought

Explore multiple reasoning paths/branches before choosing best.

Multi-Agent Prompting

Different agents/roles handle different subtasks.

Example:

  • one agent summarizes
  • one critiques
  • one verifies

14) High-Yield Compare / Contrast

SVM vs KNN vs Trees vs RF

  • SVM: margin-based classifier
  • KNN: nearest points vote
  • Decision Tree: rule-based splits
  • Random Forest: many trees combined

Perceptron vs MLP vs Deep Net

  • Perceptron: single layer
  • MLP: one hidden layer
  • Deep Net: multiple hidden layers

CNN vs RNN vs Transformer

  • CNN: spatial/local patterns
  • RNN/LSTM: sequential dependence over time
  • Transformer: attention-based sequence modeling

Overfitting vs Underfitting

  • Overfitting: too complex, memorizes
  • Underfitting: too simple, misses real pattern

Precision vs Recall

  • Precision: avoid false positives
  • Recall: avoid false negatives

17) Likely Short-Answer Explanations

Why use validation data?

To tune hyperparameters without contaminating the final test evaluation.

Why does overfitting happen?

Model is too complex or trained too aggressively relative to data.

Why ReLU is popular?

Simple, fast, helps mitigate vanishing gradients compared to sigmoid.

Why use dropout?

Improves generalization by preventing co-adaptation of neurons.

Why attention matters?

Lets model weigh the importance of different tokens dynamically.

Why positional encoding is needed?

Transformer has no recurrence/convolution, so order must be injected.
